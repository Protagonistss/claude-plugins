# Skill: 时间管理

这个技能使代理能够帮助用户优化时间使用和提高工作效率。

## 技能描述

时间管理技能使 TaskAssistant 能够：
- 评估任务所需时间
- 建议最佳工作时段
- 应用番茄工作法
- 识别时间浪费点
- 优化日程安排

## 核心方法

### 1. 番茄工作法（Pomodoro Technique）

**工作周期**：
- 🍅 工作 25 分钟（一个番茄钟）
- ☕ 休息 5 分钟
- 🍅 工作 25 分钟
- ☕ 休息 5 分钟
- 🍅 工作 25 分钟
- ☕ 休息 5 分钟
- 🍅 工作 25 分钟
- 🏖️ 长休息 15-30 分钟

**适用场景**：
- 需要高度专注的任务
- 容易分心的工作环境
- 长时间编码或写作

### 2. 时间块（Time Blocking）

将一天分为固定时间块：

```
09:00-10:30 - 深度工作（最重要任务）
10:30-10:45 - 休息
10:45-12:00 - 协作时间（会议、讨论）
12:00-13:00 - 午餐休息
13:00-15:00 - 深度工作（次要任务）
15:00-15:15 - 休息
15:15-17:00 - 浅度工作（邮件、杂务）
17:00-17:30 - 总结和计划
```

### 3. 艾森豪威尔矩阵

按重要性和紧急性分类任务：

```
            紧急          不紧急
重要    ┌─────────┬─────────┐
        │  立即做  │  计划做  │
        │  (Crisis)│(Planning)│
        ├─────────┼─────────┤
不重要  │  委托做  │  不要做  │
        │(Delegate)│(Eliminate)│
        └─────────┴─────────┘
```

### 4. 两分钟规则

如果一个任务可以在 2 分钟内完成，立即去做，不要推迟。

## 时间估算方法

### 三点估算法

对于每个任务，估算三个时间：

```
乐观时间（O）：最理想情况下的时间
正常时间（M）：通常需要的时间
悲观时间（P）：最坏情况下的时间

预期时间 = (O + 4M + P) / 6
```

**示例**：
```
任务：实现用户登录API
- 乐观时间：1小时（一切顺利）
- 正常时间：2小时（常规速度）
- 悲观时间：4小时（遇到问题）
- 预期时间：(1 + 4×2 + 4) / 6 = 2.2小时
```

## 使用示例

### 场景 1：安排一天工作

**用户**：我今天有这些任务，帮我安排一下

**代理使用此技能**：

```
# 今日工作计划

## 早晨 - 黄金时段（09:00-12:00）
✅ 专注：实现核心功能（2小时）
   - 最重要的任务安排在精力最充沛时
   - 建议：🍅🍅🍅🍅（4个番茄钟）

☕ 休息 15分钟

✅ 代码审查（1小时）
   - 需要专注但强度略低
   - 建议：🍅🍅（2个番茄钟）

## 下午 - 协作时段（13:00-17:00）
✅ 团队会议（1小时）
✅ 回复邮件和消息（30分钟）
✅ 更新文档（1小时）
✅ 处理小Bug（1.5小时）

## 晚上 - 收尾（17:00-17:30）
✅ 总结今日进展
✅ 规划明日任务
```

### 场景 2：任务时间评估

**用户**：写一个RESTful API大概要多久？

**代理使用此技能**：

```
# 时间评估：开发RESTful API

## 任务分解
1. 设计API接口（路由、参数）- 1小时
2. 实现CRUD操作 - 3小时
3. 添加验证和错误处理 - 1.5小时
4. 编写API文档 - 1小时
5. 单元测试 - 1.5小时
6. 集成测试 - 1小时

## 总计
- 纯开发时间：9小时
- 加上缓冲（20%）：10.8小时
- 建议分配：2个工作日

## 时间分配建议
第一天：
- 上午：设计 + 开始实现
- 下午：完成CRUD + 验证

第二天：
- 上午：文档 + 测试
- 下午：调试和优化
```

## 生产力技巧

### 1. 深度工作时段

识别和保护深度工作时间：
- 🌅 早晨通常精力最充沛
- 🔕 关闭通知，减少干扰
- 🎯 一次只专注一个任务
- ⏰ 设定明确的时间界限

### 2. 批量处理

将类似任务集中处理：
- 📧 集中时间处理邮件（而非随时查看）
- 💬 集中时间回复消息
- 📝 集中时间更新文档

### 3. 精力管理

根据精力水平安排任务：
```
高精力任务：复杂编码、架构设计
中精力任务：代码审查、调试
低精力任务：整理文档、回复邮件
```

### 4. 避免多任务

研究表明多任务切换会：
- ⏱️ 浪费 40% 的生产力
- 🧠 增加认知负担
- 😫 导致更多错误

**建议**：一次只做一件事，做完再做下一件。

## 时间追踪

建议追踪的数据：
- ⏱️ 任务实际耗时 vs 预估耗时
- 🎯 高效工作时段
- 🚫 被打断的次数和原因
- ✅ 每日完成的任务数

## 应用原则

1. **现实可行**：估算要符合实际能力
2. **留有余地**：总是预留缓冲时间
3. **定期反思**：回顾时间使用情况
4. **持续优化**：根据实际情况调整方法

## 集成说明

这个技能自动被 TaskAssistant 代理使用。当涉及时间规划、任务排期等话题时会自动激活。
