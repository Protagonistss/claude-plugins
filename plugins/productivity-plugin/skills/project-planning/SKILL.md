# Skill: 项目规划

这个技能使代理能够帮助用户进行项目规划和任务分解。

## 技能描述

项目规划技能使 TaskAssistant 能够：
- 分析项目需求和目标
- 将项目分解为可管理的任务
- 创建合理的时间表
- 识别项目风险和依赖关系
- 制定项目里程碑

## 使用场景

### 1. 新项目启动

当用户开始一个新项目时，这个技能帮助：
- 定义项目范围
- 识别主要交付物
- 创建初始任务列表
- 设定项目时间线

### 2. 任务分解

对于复杂任务，这个技能提供：
- 逐层分解策略
- 识别原子任务
- 估算任务工作量
- 确定任务优先级

### 3. 风险评估

帮助识别：
- 技术风险
- 资源约束
- 时间压力
- 依赖关系

## 核心能力

### 能力 1：任务分解

将复杂项目分解为：
```
项目
├── 阶段 1
│   ├── 任务 1.1
│   ├── 任务 1.2
│   └── 任务 1.3
├── 阶段 2
│   ├── 任务 2.1
│   └── 任务 2.2
└── 阶段 3
    └── 任务 3.1
```

### 能力 2：时间估算

为每个任务提供：
- 乐观估计（最快完成时间）
- 正常估计（常规完成时间）
- 悲观估计（最坏情况时间）
- 推荐时间（综合评估）

### 能力 3：依赖管理

识别任务间的依赖：
- 前置任务（必须先完成的）
- 并行任务（可以同时进行的）
- 后续任务（依赖当前任务的）

### 能力 4：里程碑设定

创建项目检查点：
- 阶段性目标
- 交付时间
- 评审节点
- 验收标准

## 技能参数

```json
{
  "granularity": "medium",
  "timeUnit": "hours",
  "considerRisks": true,
  "includeDependencies": true,
  "maxDepth": 3
}
```

## 风险提示
⚠️ 数据库设计可能需要评审，预留调整时间
⚠️ UI设计可能需要多轮迭代
⚠️ 集成测试可能发现未知问题

## 最佳实践

1. **自上而下分解**：从大到小，逐层细化
2. **合理粒度**：任务不宜太大（>1天）或太小（<1小时）
3. **时间缓冲**：预留10-20%的缓冲时间
4. **定期复审**：项目进行中定期更新计划
5. **灵活调整**：根据实际情况调整任务和时间

## 集成说明

这个技能自动被 TaskAssistant 代理使用，无需额外配置。当用户提及项目规划、任务分解等关键词时，代理会自动激活此技能。
